# Agent Spec: Ambient Expense Agent

## Overview
An ambient, event-driven agent that automatically processes expense reports. It parses incoming JSON events (plain or base64 encoded for Pub/Sub), evaluates the amount against a configurable threshold, and conditionally routes the request.

## Architecture
Built using ADK 2.0 Graph Workflow API.
- **parse_event**: Decodes and parses the JSON event into an `ExpenseReport` schema.
- **auto_approve**: Instantly approves expenses under $100.
- **risk_reviewer**: For expenses $100+, uses an LLM (`gemini-3.1-flash-lite`) to assess risk factors.
- **human_review**: Human-in-the-loop step (`RequestInput`) that pauses the workflow, presents the LLM's assessment, and waits for a decision.
- **record_outcome**: Final node that converges both paths and records the outcome.

## Tools Required
- None currently (pure workflow logic).
- Future: Database integration to actually record the outcome, or notification service to alert the submitter.

## Constraints & Safety Rules
- Must use python code logic for routing threshold, NOT an LLM prompt.
- LLM is strictly used for risk factor evaluation.
- All expenses $\ge$ 100 MUST block for human approval.

## Success Criteria
- Base64 and plain JSON events are successfully parsed.
- $\lt$ 100 routes completely autonomously.
- $\ge$ 100 routes to LLM then correctly pauses for human input.
- Workflow resumes properly with human decision.
